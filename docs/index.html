PhotonLab

Fast, non-destructive photo editor for Android and Linux/Windows desktop. All edits are applied in real time — the original image is never modified.

v2.0.1 Android Linux Windows

Overview

PhotonLab is a photo editor focused on speed and simplicity. It offers a complete set of tone, color, and creative tools with a clean dark interface. The same edit pipeline runs on Android (via AGSL shaders) and desktop (via LWJGL/OpenGL), so results are identical across platforms.

Non-destructive

Every adjustment is stored as parameters in an EditState. The source image is never overwritten — export creates a new file.

Real-time preview

Changes are rendered immediately. A quick 512 px preview appears instantly; a full-resolution preview follows within 250 ms.

Batch processing

Open multiple images at once, edit each independently, and export them all with a single tap. Each image remembers its own settings.

Full resolution export

Export renders at the original source resolution with no downsampling, regardless of preview zoom level.

Installation

Linux (.deb — Ubuntu / Debian)

  1. Download photonlab_2.0.1_amd64.deb from the GitHub Releases page.
  2. Double-click the file in your file manager, or install via terminal:
    sudo dpkg -i photonlab_2.0.1_amd64.deb
  3. If dependency errors appear: sudo apt-get install -f
  4. Launch from your application menu, or run photonlab in a terminal.
Requires a 64-bit system. Compatible with Ubuntu 20.04 LTS and later, and equivalent Debian versions.

Windows

Build from source using buildExe.bat (requires JDK 21 and Gradle). A pre-built .exe installer will be available in a future release.

Android

Install the APK directly or via Google Play (when available). Requires Android 8.0 (API 26) or later.

Opening Images

Single image

Click the folder icon in the top bar (or drag and drop a file onto the window on desktop). PhotonLab supports JPEG, PNG, HEIC, WebP and all formats your platform's image loader supports.

Batch (multiple images)

In the file picker, select multiple image files. PhotonLab loads them as a batch. Navigate between images using the arrow buttons in the top bar or by swiping left/right on the image.

Each image in a batch remembers its own edit settings. Switching images saves the current state automatically.

Drag & Drop Desktop

Drop one or more image files directly onto the PhotonLab window to open them.

Tone Adjustments

Found in the TONE tab. These sliders affect the overall brightness and contrast of the image.

ParameterRangeDefaultDescription
Exposure−5 to +5 EV0Multiplicative brightness on linear light — mimics camera exposure. Each +1 doubles the brightness.
Luminosity−100 to +1000Additive brightness offset applied after exposure. Lighter touch than exposure.
Contrast−100 to +1000S-curve contrast. Positive values make darks darker and lights lighter.
Highlights−100 to +1000Adjusts only bright tones. Pull down to recover blown highlights.
Shadows−100 to +1000Adjusts only dark tones. Push up to lift shadows without affecting highlights.
Sharpening0 to 1000Unsharp mask sharpening. Increases local contrast along edges.
Noise / Grain−100 to +1000Negative: multi-pass Gaussian denoise. Positive: procedural film grain (luminance + chroma).
Double-click any slider to reset it to its default value.

Color Adjustments

Found in the COLOR tab.

ParameterRangeDefaultDescription
Saturation−100 to +1000Adjusts HSL saturation of all colors equally. −100 produces a grayscale image.
Vibrance−100 to +1000Intelligent saturation: boosts muted colors more than already-vivid ones. Gentler than Saturation.
Temperature−100 to +1000Shifts the color balance cool (blue) ↔ warm (orange). Simulates white balance adjustment.
Tint−100 to +1000Shifts the color balance green ↔ magenta. Used alongside Temperature for accurate white balance.

LUT (Color Grading)

Found in the LUT tab. LUTs (Look-Up Tables) apply a pre-defined color grade to your image — useful for cinematic looks, film emulation, and consistent cross-image grading.

Supported formats

  • .cube — 3D LUT files (industry-standard, any grid size)
  • HaldCLUT PNG — Hald Color Look-Up Tables stored as PNG images

Importing a LUT

  1. Go to the LUT tab.
  2. Click Import LUT and select a .cube or HaldCLUT PNG file.
  3. The LUT is applied instantly and shown as the active LUT.
  4. Click the ✕ button next to the LUT name to remove it.
On desktop, you can set a LUT folder in Settings so the file picker opens there by default.

LUT in presets

When saving a preset, you can include the current LUT so it's applied automatically when the preset is loaded.

Transform

Found in the TRANSFORM tab. Transform settings are applied before tone and color adjustments.

Crop

Click Crop Image in the TRANSFORM tab to open the full-screen interactive crop editor.

Aspect ratio options

  • Free — drag handles to any shape
  • 1:1 — square
  • 4:3 — classic photo / screen
  • 3:2 — standard DSLR sensor
  • 9:16 — portrait / mobile screen

The crop is stored as normalized coordinates (0–1) and applied non-destructively. The full image remains available until export.

Found in the TRANSFORM tab. Adds a solid-color border around the image.

Date Imprint

Found in the TRANSFORM tab. Burns a film-style date stamp onto the image, drawn from the photo's EXIF metadata. Applied before tone grading for an authentic film-camera look.

If the image has no EXIF date, the stamp will show "No EXIF date" in the preview. Make sure your source files contain EXIF date information.

Grid Overlay

Click the grid icon in the top-right of the image area to cycle through overlay modes:

  • None — no overlay
  • Rule of Thirds — 3×3 composition guide
  • Ninths — 9×9 dense grid
  • Golden Ratio — guides at φ ≈ 0.618 and 1−φ ≈ 0.382

The grid is constrained to the image bounds and transforms with zoom and pan.

Histogram

Click the histogram icon in the top-right of the image area to toggle the RGB histogram overlay. It displays in the bottom-left corner of the image.

  • Channels: Red, Green, Blue
  • Logarithmic scale — reveals detail in both highlights and shadows
  • Semi-transparent overlay — does not obstruct the image
  • Updates live as you move sliders

Compare Original

Press and hold on the image to temporarily display the unedited original. Release to return to the current edit.

A small ORIGINAL label appears in the corner while comparing. The original is shown at the same zoom and pan position as the current edit.

Panel Layouts Desktop

Two layout modes are available on desktop, selectable in Settings. The choice is saved and remembered across sessions.

Bottom (default)

The image fills the entire window. The tool panel overlays from the bottom. Drag the handle at the top of the panel to resize it.

Panel height: 48 dp min · 600 dp max · 340 dp default.

Side

The image occupies the left side; the tool panel is on the right. Drag the vertical divider to resize.

Panel width: 240 dp min · 520 dp max · 320 dp default.

Edit History

PhotonLab maintains a full undo/redo history for every image.

  • Undo / Redo buttons in the top bar
  • History dialog: click the history icon to see a list of all states. Click any entry to jump directly to that point.
  • Continuous slider drags are compressed into a single history entry (so undo steps over whole adjustments, not individual pixel changes).

Presets

Presets let you save a group of adjustments and apply them to other images in one click.

Saving a preset

  1. Open Settings → Presets → Create Preset.
  2. Enter a name and choose which parameters to include.

Included parameters

Any combination of: Exposure, Luminosity, Contrast, Highlights, Shadows, Saturation, Vibrance, Temperature, Tint, Sharpening, Noise, Rotation, LUT, Frame, Date Imprint.

Applying to a batch

When applying a preset to a batch image that hasn't been edited yet, you'll be asked whether to auto-apply the same preset to all remaining unedited images.

Managing presets

  • Apply — applies the preset to the current image
  • Rename — renames the preset
  • Delete — removes the preset (with confirmation)

Batch Processing

Open multiple images with the file picker (or drag multiple files onto the window). PhotonLab loads them all into a batch queue.

Navigating images

  • Use the ← → buttons in the top bar.
  • Swipe left/right on the image (swipe at least 80 dp to trigger navigation).
  • A progress indicator shows "N / M" (current / total).

Per-image settings

Each image remembers its own edit settings independently. Switching away from an image saves its state automatically.

Reset an image

Press the Reset button (↺ icon) in the top bar to clear all edits for the current image.

Export

ButtonAction
ExportSaves the current image with all edits applied.

Output

  • Format: JPEG
  • Quality: Configurable 50–100% (default 95%). Set in Settings.
  • Resolution: Full source resolution — no downsampling.
  • Filename: photonlab_<timestamp>.jpg
  • Location: ~/Pictures/PhotonLab/ (desktop) or device gallery (Android).
A snackbar notification shows the saved file path after each successful export.

Background Export

If you try to close PhotonLab while exports are still processing, a dialog appears with three options:

OptionBehavior
Continue in backgroundThe window closes (or hides). PhotonLab continues processing all queued exports silently and exits automatically when the last export is done.
WaitThe dialog stays open. PhotonLab finishes all exports while you watch the count go down, then you can close normally.
Close anywayForce-closes immediately. Any remaining queued exports are cancelled.
Using Close anyway will discard unfinished exports. Use Continue in background to safely complete all work.

Desktop (Linux / Windows)

System requirements

  • 64-bit OS — Ubuntu 20.04+ or equivalent Debian, Windows 10+
  • OpenGL 3.3 compatible GPU (used for tone rendering acceleration)
  • JRE 21 is bundled — no separate Java installation needed
  • ~200 MB disk space

GPU-accelerated rendering

Tone adjustments (exposure, contrast, saturation, etc.) are processed via an LWJGL offscreen OpenGL context using custom GLSL shaders. A CPU fallback is used automatically if OpenGL is unavailable.

Adaptive preview resolution

PhotonLab uses a two-phase rendering strategy:

  1. Quick preview (512 px): rendered immediately when any parameter changes.
  2. Full preview (1920 px × zoom factor): rendered 250 ms after the last change. Zooming in triggers a higher-resolution render automatically.

Linux icon & taskbar

PhotonLab uses three mechanisms to ensure the correct icon appears everywhere:

  • X11 WM_CLASS set to photonlab for correct taskbar grouping
  • StartupWMClass=photonlab in the .desktop file for GNOME taskbar matching
  • AppStream metadata bundled in the .deb for the correct icon in GNOME Software

Android

Requirements

  • Android 8.0 (API 26) or later
  • ~50 MB storage

Permissions

  • READ_MEDIA_IMAGES — Android 13+ (API 33+)
  • READ_EXTERNAL_STORAGE — Android 8–12 (API 26–32)

Rendering

On Android 13+ (API 33+): AGSL RuntimeShader pipeline for GPU-accelerated rendering. On older devices: automatic CPU fallback via RenderScript Compat.

Export location

Exported files are saved to the device gallery via MediaStore. No WRITE_EXTERNAL_STORAGE permission required on Android 10+.

Background export

Pressing back while exports are in progress shows the same three-option dialog as desktop. Continue in background moves the app to the background; it finishes processing and calls finish() automatically when done.

Version History

v2.0.1
March 21, 2026
  • Fixed .deb installation failure on older Ubuntu/Debian — replaced libasound2t64 and libpng16-16t64 hard dependencies with OR alternatives that also accept the non-t64 variants.
  • Removed the 96 px upper cap on date imprint text size — the stamp now scales freely based on the size percentage, with only a 6 px minimum enforced.
  • Style selector for date imprint changed from a horizontally scrollable chip row to a compact dropdown menu.
  • Pending export counter now reflects all queued jobs (not just the one currently processing), so "Continue in background" correctly waits for the full batch.
  • Fixed "Continue in background" on close: the process now stays alive until every export in the queue is complete, then exits automatically.
